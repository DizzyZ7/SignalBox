# Public Launch Checklist

This checklist helps prepare SignalBox for an open-source public launch.

## Repository

- [ ] Clear GitHub description.
- [ ] Useful GitHub topics.
- [ ] License is visible.
- [ ] Contribution guide is visible.
- [ ] Issue templates are enabled.
- [ ] Pull request template is enabled.
- [ ] Documentation index is easy to find.

## README

- [ ] Clear product positioning.
- [ ] Short problem statement.
- [ ] Feature list with practical value.
- [ ] Five-minute quickstart.
- [ ] Links to core docs.
- [ ] Links to roadmap and comparison.
- [ ] Screenshots or GIFs.

## Documentation

- [x] Getting started guide.
- [x] Self-hosting guide.
- [x] Environment reference.
- [x] Recipes.
- [x] Roadmap.
- [x] Comparison guide.
- [x] Contributing guide.
- [x] Documentation index.
- [ ] Screenshots guide.
- [ ] Demo script.

## Product readiness

- [x] Event storage.
- [x] Deduplication.
- [x] Replay.
- [x] Delivery queue.
- [x] Telegram delivery.
- [x] HTTP forwarding.
- [x] Admin UI.
- [x] Admin audit log.
- [x] Source incident snapshot.
- [x] Metrics endpoint.
- [x] Backup and restore scripts.

## Quality gates

- [x] Go tests in CI.
- [x] Go vet in CI.
- [x] Build in CI.
- [x] Shell checks in CI.
- [ ] Admin UI JavaScript syntax check.
- [ ] Example payload validation.
- [ ] Basic smoke test.

## Launch assets

- [ ] Screenshots.
- [ ] Short demo GIF or video.
- [ ] Public launch post.
- [ ] Demo instance.
- [ ] Cloud waitlist page.

## First users

- [ ] Find early testers.
- [ ] Track installation friction.
- [ ] Track missing providers.
- [ ] Turn repeated questions into docs.
- [ ] Publish small releases regularly.
