# Code of Conduct

SignalBox is an open-source project for people who want to build, operate and improve webhook infrastructure.

The goal is to keep the project useful, technical and respectful.

## Expected behavior

Participants are expected to:

- be respectful in issues, discussions and pull requests;
- keep feedback technical and actionable;
- explain trade-offs instead of attacking people;
- assume good intent when possible;
- help keep the project welcoming to new contributors;
- avoid sharing private data, secrets or customer payloads in public threads.

## Unacceptable behavior

The following behavior is not acceptable:

- harassment or personal attacks;
- discriminatory language;
- threats or intimidation;
- deliberate disruption of technical discussions;
- publishing private information without permission;
- posting secrets, tokens or sensitive payloads in public issues.

## Enforcement

Project maintainers may:

- edit or remove inappropriate comments;
- close or lock unproductive issues;
- ask participants to move sensitive reports to a private channel;
- block users who repeatedly violate this policy.

## Reporting

For security issues, follow [`SECURITY.md`](SECURITY.md).

For behavior concerns, contact the maintainer through the GitHub profile listed in the repository.

## Scope

This code of conduct applies to project repositories, issues, pull requests, documentation, examples and public project discussions.
